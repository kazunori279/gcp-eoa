## Notable: the headless validation caught a real bug

During Q2, `compute_reroute` raised a Python `heapq` comparison error — when multiple routes
shared identical departure/arrival times, the heap fell back to comparing the dict payloads.
agy diagnosed it, fixed `app/tools.py` (added a tiebreaker so dicts are never compared), and
re-ran cleanly. This is exactly the value P3 adds: a headless, repeatable validation that
exercises the real tool code and exposes edge-case bugs the happy-path demo would miss.
