## The Step 2 caveat (the main finding)

The prompt instructs the agent to **expect Agent Identity to be opt-in and OFF** — i.e. a
default service agent `service-PROJECT_NUMBER@gcp-sa-aiplatform-re.iam.gserviceaccount.com`,
"not a SPIFFE principal." But **M2 deploys with `agents-cli deploy --agent-identity`**, which
turns Agent Identity **ON**. So the engine's real `effectiveIdentity` this run was a
**SPIFFE-attested** identity:

```
agents.global.org-…system.id.goog/resources/aiplatform/projects/378474938894/locations/us-east1/reasoningEngines/5470642094232043520
```

agy correctly observed the SPIFFE identity — but the prompt's stated expectation is wrong for
the workshop's own flow, and the least-privilege role audit it directs
(`--filter="bindings.members:service-PROJECT_NUMBER@gcp-sa-aiplatform-re…"`) targets the
**default service agent**, not the SPIFFE runtime identity that is actually in effect. So the
audit may be inspecting the wrong principal.

## Improvement plan for `m3.html`

- **P1 — Fix the Step 2 identity expectation to match M2.** Because M2 deploys with
  `--agent-identity`, Agent Identity is **ON** and `effectiveIdentity` is a **SPIFFE**
  principal. Update the prompt to expect that (and explain it as the *governed* outcome), or
  explicitly branch ("if you deployed with `--agent-identity`, expect a SPIFFE identity;
  otherwise the default service agent"). As written it tells students to expect the opposite
  of what they built.
- **P2 — Point the least-privilege audit at the right principal.** With Agent Identity on, the
  IAM check should reason about the SPIFFE runtime identity (or clearly explain why the default
  service agent's roles are still the relevant grant). Today the `--filter` targets the default
  service agent, which may not be the identity actually executing the agent.
- **P3 — (Minor) `transit-shield` region note.** Step 3 hardcodes `us-central1` for Model Armor
  (independent of the agent's `us-east1`) and says so — that's fine and worked; just keep the
  "need NOT match the agent region" note prominent so students don't think it's a mistake.
