## Improvement plan for `m5.html`

- **P1 — Make AGY actually write `registration_results.md`.** It's a requested deliverable and
  was skipped. Either tighten the wording ("create the file `registration_results.md` with the
  registered agent resource name + both console links") or drop the requirement if it's not
  essential. As-is, the prompt asks for an artifact the agent doesn't reliably produce.
- **P2 — Flag the GE UI test as the human/browser step it is.** `cmd-gemini-test` needs the
  Gemini Enterprise app UI; make explicit that this is a manual browser check (it already
  reads that way, but the workshop should set the expectation that publishing is CLI-verified
  while the end-user query is UI-verified).
- **P3 — (Optional) add a CLI post-publish check.** A `agents-cli publish gemini-enterprise
  --list` (or a discoveryengine GET on the registered agent) after publishing would give a
  headless confirmation that the agent is registered, complementing the browser test.
