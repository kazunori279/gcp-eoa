## Fixes applied (from the first-run improvement plan) — all validated this run

- **P1 — Simplified Step 1 (overwrite, not merge).** `mcp_config.json` ships as a 0-byte file
  in the lab, so the prompt now **overwrites** it (nothing to preserve) and only **merges**
  `settings.json` (which has real keys). The ~40% of defensive "treat empty as `{}`" prose is
  gone. Validated: install + native `answer_query` worked from the genuine fresh-lab state.
  _(The "fix the lab image" option wasn't available, so we shrank the prompt instead.)_
- **P2 — One self-healing download.** The two copy-paste blocks (official + mirror fallback)
  are merged into a single prompt using `curl -fL … || curl -fL <mirror>` (the `-f` makes the
  fallback fire on HTTP errors). Removes a manual student decision. Validated.
- **P3 — Removed the `dk.py` fallback callout** from the page (the helper stays in the repo as
  an optional escape hatch). Native MCP is the happy path; students no longer pre-empt it.
- **P4 — ADC identity callout** added to Step 3: `setup` runs as the **workstation service
  account** via ADC (not the student login), which M2/M3 reason about later.
- **plan.md template** updated so M2 lists the **6-step deploy-once** flow (Prep → Sessions →
  Code-exec → Smoke test → Deploy once → Test). Validated in the generated `plan.md`.
