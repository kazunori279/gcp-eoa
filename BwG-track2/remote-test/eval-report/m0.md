## Key finding (validated, not a bug): the AGY restart is load-bearing on a fresh lab

On a genuinely fresh lab the `agy` session starts **before** the DK MCP config
exists (the lab ships a 0-byte `mcp_config.json`, which silently disables every
MCP tool). So the DK MCP only becomes available **after** restarting AGY тАФ and
`cmd-dk-tryit` will fail in the same session if the restart is skipped. The
workshop content already instructs this restart (Step 1 point 5 + the "Then
restart тАФ run `agy`" callout), and this run confirms it is **necessary**, not
optional, on a clean lab. The restart cost no continuity: `plan.md` doesn't
exist until Step 2, so nothing is lost.

> Harness note: prior reports described M0 as "one continuous conversation."
> That holds only on a re-used lab where `mcp_config` was already populated when
> the session started. On a true fresh lab the restart between Step 1 and the
> try-it is required тАФ which is exactly what the page tells the student to do.
