The Quality Flywheel Concept

Quality improvement for agents is a loop, not a one-time check:

  1. Evaluate — run the agent against many scenarios and score it
  2. Analyze — find loss clusters (groups of failures with a common cause)
  3. Optimize — fix the biggest cluster (often by refining the system instruction)
  4. Repeat — each turn raises quality empirically

This is how you go from "it works on my one test query" to "it works on every query."

Simulate the Crisis at Scale

We need to test the agent against many different scenarios, not just one query. User simulation generates diverse commuter questions; environment simulation injects edge cases like the disruption feed going down.

Paste into AGY Generate 100 commuter scenarios during a St Pancras signal failure and run them against my agent, including a case where the disruption feed is unavailable.

Expected Result

AGY runs agents-cli eval and generates:

  • 100 synthetic commuter queries in different styles (direct, intent-based, vague, emotional)
  • Edge case environments (feed unavailable, empty data, multiple disruptions)
  • Traces collected for every scenario

What just happened: The eval skill orchestrated simulation at scale. 100 scenarios catch failures your one test query never will. See the evaluation dashboard in the next step.

6 min

Find the Weak Spots & Fix Them

The system groups failures by common cause into loss clusters. Example: "fails when origin IS the disrupted station" or "fails when the commuter asks a vague question." We'll auto-optimize the system instruction to fix the biggest cluster.

Paste into AGY Analyze the failures and optimize my system instruction to fix the biggest cluster.

Expected Result

AGY runs adk optimize / Optimizer service and:

  1. Identifies the largest loss cluster (e.g., "fails when origin IS the disrupted station")
  2. Generates a refined system instruction targeting that failure mode
  3. Re-runs the failed scenarios to verify improvement
  4. Shows before/after scores

What just happened: Quality improvement is empirical and automatable, not guesswork. One refined instruction, targeting the top failure cluster, raises quality across the board. See the The Flywheel tab for the full pattern.

Mentor Checkpoint — Done When:

  • 100 simulated scenarios have been run against your agent
  • Evaluation dashboard shows scores for all metrics
  • You've identified at least one loss cluster
  • System instruction has been optimized with a before/after score improvement

The Quality Flywheel

The Quality Flywheel is the official loop for agent quality improvement. Each revolution raises quality empirically.

Why This Works

  • Simulation creates coverage — 100 scenarios catch failures your one test query never will
  • AutoRaters remove subjectivity — metrics are defined, measured, compared
  • Loss clusters reveal patterns — not random failures, but systematic weaknesses
  • Automatic optimization closes the loop — the system proposes and tests a fix, not you
