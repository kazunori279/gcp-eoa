Agent Context Test Plan

Validate that workshop context snippets are sufficient for a coding agent to build the transit-crisis ADK agent end-to-end
What are we testing?
Whether a coding agent (simulating Antigravity) can build a complete, syntactically valid ADK transit-crisis agent using only the agent context snippets that workshop participants copy-paste from the HTML pages.

Methodology

Launch an independent subagent with the exact text from 4 context blocks — the same content participants will paste into Antigravity. The subagent:

  1. Generates synthetic GTFS data — 17 stations, ~20 trips, 5 routes, plus disruptions.json
  2. Scaffolds the ADK project — agent/ directory with __init__.py, agent.py, tools.py, and pyproject.toml
  3. Implements 3 tools — get_scheduled_departures, check_disruptions, compute_reroute
  4. Writes the system instruction — reconciliation rules per the context
  5. Self-validates — Python AST parse to confirm no syntax errors

What this reveals

  • Missing info — Where the subagent had to guess or make assumptions not covered by the context blocks
  • Ambiguities — Where the subagent made reasonable but potentially wrong choices
  • Structural issues — Where generated code doesn't follow the expected project layout

Each gap becomes a TODO for improving the workshop HTML before the event.

Constraint
The subagent does NOT install dependencies or run the agent. It generates code and validates syntax only. The test proves instructional sufficiency, not runtime correctness.

Smoke Test Results

Functional smoke tests ran directly on the generated tools with the synthetic data:

  • get_scheduled_departures("St Pancras", "13:00", "17:00") → 6 departures found, including EUR9005 at 13:31 (the affected train)
  • check_disruptions(station_name="St Pancras") → 4 disruptions: EUR9005 cancelled, EUR9014 cancelled, EUR9012 delayed +45min, EUR9007 delayed +30min
  • compute_reroute("St Pancras", "Paris Gare du Nord", "13:00") → skipped cancelled EUR9005, found route via delayed EUR9012 to Lille then EUR9007 to Paris, arriving 17:51, 1 change, 287 min total

The test query "My 13:31 St Pancras to Paris service is affected" matches trip EUR9005 (13:31 departure, cancelled). The reroute correctly avoids it and finds a viable 2-train alternative through Lille.

Recommendations for Workshop Materials

Based on the assumptions the subagent had to make, consider these remaining improvements:

  1. Model name — FIXED. Added "do not change it" note to m1 context blocks. Subagent now correctly uses gemini-3.5-flash
  2. Add ADK export convention to m1 context: mention that __init__.py should export root_agent
  3. Clarify "remove disrupted" in compute_reroute spec: specify whether to exclude only cancelled trips or also delayed ones
  4. Specify reroute algorithm metric: "earliest arrival" vs "fewest changes" — the subagent chose Dijkstra by earliest arrival, while BFS would find fewest-stops routes
  5. Define disruptions.json schema explicitly: single event with affected_trips[] array, or array of independent disruptions?

Items 2-5 are low severity — the subagent made reasonable choices in all cases. The context blocks are strong enough for a successful build.
